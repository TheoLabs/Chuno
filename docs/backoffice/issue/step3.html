BO Step 3 · 모더레이션 · 신고 · 파일

신고 접수·처리 큐와 부적절 파일 정리. 신고 도메인 신설 여부는 열린 질문. (총 4 이슈, 전부 미착수)

BO3-1신고 · 모더레이션 도메인 신설 여부 결정/설계DESIGNS
Report 엔티티를 신설할지, MVP 이후로 보류할지부터 결정한다.
  • Report 엔티티 신설 여부 — 열린 질문
  • 대상 타입·사유·상태(OPEN·RESOLVED)
  • 사용자단 신고 접수 진입점 필요 여부
완료 — 신고 도메인 설계안 또는 "MVP 이후 보류" 결론이 문서화된다.
선행: BO1-1관련MVP도메인

BO3-2신고 접수 + 관리자 신고 큐/처리 APIBACKENDM
사용자 신고를 접수받아 관리자 큐에서 처리하고, 필요 시 제재로 연계한다.
  • (필요시) 사용자 POST /reports
  • GET /api/admins/reports 큐
  • PUT /api/admins/reports/:id/resolve
  • 처리 시 제재(BO2-3) 연계
  • 감사 기록
완료 — 신고 접수 → 관리자 처리 → (선택) 제재 연계가 동작한다.
선행: BO3-1, BO1-3, BO2-3관련MVP

BO3-3파일 / 미디어 모더레이션 APIBACKENDS
부적절한 업로드 파일을 관리자가 조회·삭제한다.

  • OPERATOR 이상 role
  • 감사 기록
