백오피스 MVP 로드맵/BO Step 1

BO Step 1 · IAM 기반 + 법적문서 관리 재스코프

관리자 계정·역할·감사로그 도메인을 세우고, 사용자 트랙의 "관리자 authz" 빈틈을 실 role-RBAC로 메운다.

목표 — 관리자가 외부 SSO로 로그인해 role에 맞는 관리자 API만 호출할 수 있고, 법적문서 초안 등록·예약 활성화가 관리자 전용 API로 재스코프된다.

🎫 이 스텝의 상세 이슈 (7) →

선행: 없음 (시작점)
DESIGN Admin & IAM 도메인
신규 "Admin & IAM" 컨텍스트 + AdminAccount·Role·AdminAuditLog 엔티티 카드 — BO1-1
role-only RBAC 불변식 — Permission 엔티티 없음, Role(SUPER_ADMIN·OPERATOR·SUPPORT)이 역할 단위 게이팅 경계 — BO1-1
감사로그 append-only + 논리참조 규칙, ERD 반영 — BO1-1
BACKEND 관리자 신원 · 인증 · 감사
AdminAccount·Role 엔티티 + 리포지토리 + seed — 외부 IdP 신원(provider+subject, passwordHash 없음), UNIQUE(provider, subject), super admin seed 직접 주입 — BO1-2
관리자 SSO 인증 + AdminGuard + role 게이팅 — /api/admins/auth/login·refresh·logout, 별도 JWT 시크릿, @RequireRoles(...) — BO1-3
AdminAuditLog 기록 인프라 — 감사 인터셉터 + GET /api/admins/audit-logs — BO1-4
BACKEND 법적문서 관리 재스코프
관리자 법적문서 관리 API(S1-16 재스코프) — GET/POST/PUT /api/admins/legal-documents, PATCH 금지, OPERATOR 이상 — BO1-5
관리자 법적문서 활성화 API(S1-18에서 이전) — POST /api/admins/legal-documents/activate-due, 멱등, 내부 스케줄러 미사용 — BO1-6
UI 백오피스 프론트
프론트 스택 확정 — React + TypeScript + React Query + React Router DOM + Ant Design(별도 웹 SPA)
관리자 로그인·법적문서 관리 화면(#admin-login·#admin-legal) — BO1-7 needsWireframe: true, 어드민 와이어프레임은 후속
완료 기준 — 관리자가 외부 SSO 토큰으로 로그인해 admin JWT를 받고, role에 따라 /api/admins/* 접근이 게이팅되며, 법적문서 초안 등록·예약 활성화가 관리자 전용 API로 동작한다.
