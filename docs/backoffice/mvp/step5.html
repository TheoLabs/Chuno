BO Step 5 · 포인트 · 랭킹 · 티어 · 공지 · 지표

보상/보정 포인트, 랭킹·티어 운영, 공지 발송, 운영 대시보드 지표. 포인트 조정은 미구현 Points 컨텍스트에 의존.

목표 — 관리자가 포인트를 보정하고, 티어 컷·랭킹을 재빌드하고, 전체/세그먼트 공지를 발송하고, 핵심 운영 지표를 대시보드로 본다.

선행: BO Step 1
BACKEND 포인트 · 랭킹 · 공지 · 지표
포인트 수동 조정 API — POST /api/admins/users/:id/points(사유·refId), 잔액≥0 불변식, PointTransaction으로만 Points&Rewards 컨텍스트 구현 후속 의존 — BO5-1
랭킹 · 티어 운영 — 티어 컷 조회/변경, 랭킹 리빌드 트리거, 부정 유저 제외 — BO5-2
공지/이벤트 푸시 발송 API — POST /api/admins/notifications/broadcast(전체/세그먼트), 발송 이력, S5-1 발송기 재사용 공지 도메인 신설 여부 열린 질문 — BO5-3
운영 대시보드 지표 API — GET /api/admins/metrics(기간별 집계), 읽기 전용 S5-7(분석)과 인접, 중복 방지 보강 관계 — BO5-4
UI 백오피스 프론트
대시보드·공지·포인트 화면(#admin-dashboard·#admin-notice·#admin-points) — BO5-5 needsWireframe: true
