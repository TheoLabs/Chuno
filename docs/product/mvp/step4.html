Step 4 · 점수 · 랭킹 · 기록

경주가 끝나면 점수가 매겨지고, 누적 랭킹·기록에 반영되는 보상 루프. 리텐션의 핵심.

목표 — 경주 종료 → 4축 점수 계산 → 결과 화면 표시 → 랭킹·기록에 누적 반영까지 자동으로 이어진다.

🎫 이 스텝의 상세 이슈 (9) →

선행: Step 3(RaceFinished 이벤트) — 정산의 트리거
BACKEND 정산 체인 (RaceFinished 구독)
완료ScoringModule — 서버 권위 순수함수 4축(등수≤300·거리≤200·완주220·여유≤100, ≤1000). 배점 밸런싱은 이후, MVP는 초안 그대로 S4-1
완료RaceResult 저장 — 유저별 완주여부·거리·완주시간·등수·점수, (raceId,userId) 유니크+count 선검사로 멱등 S4-2
완료RunnerStats 누적 — totalScore·거리·경기수·승수, 티어 계산(컷 기준 임시), RaceStatApplied 마커로 멱등(S4-2와 독립된 둘째 핸들러) S4-3
완료랭킹 — 전체 누적(RunnerStats) + 주간/월간(RaceResult createdAt 7/30일 SQL 윈도우 집계, Season 테이블 없음), 내 순위·주변 ±3 S4-4 — Redis ZSET은 MVP 미도입/후속 최적화
완료엔드포인트 GET /rankings?scope=, GET /users/me/results(기록), GET /races/:id/result — 전부 UserGuard·GET만 S4-5
APP 화면 실연동
완료결과 화면 — 시상대·순위·점수 4축 분해 실데이터(완주/DNF) S4-6
완료랭킹 — 전체/주간/월간 세그먼트 실데이터, 내 랭킹 카드·티어 S4-7
완료기록 — 지난 경주 리스트 + 상세, 프로필 누적 통계(거리·경기·승률) S4-8
완료mock 데이터·레이어 제거, 전 화면 실 repository로 교체 S4-9 — flutter analyze 이슈 없음·테스트 133
완료 기준 — 경주를 마치면 즉시 결과 화면에 점수가 뜨고, 랭킹·기록·프로필 통계에 누적 반영된다. 주간/월간 랭킹이 날짜 윈도우로 맞게 집계됨. 충족 — tsc 0에러·jest 63 통과·부팅+인증 401(백엔드), flutter analyze 이슈 없음·테스트 133(앱) 확인.
