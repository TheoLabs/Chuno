MVP 로드맵/Step 4

Step 4 · 점수 · 랭킹 · 기록

경주가 끝나면 점수가 매겨지고, 누적 랭킹·기록에 반영되는 보상 루프. 리텐션의 핵심.

목표 — 경주 종료 → 4축 점수 계산 → 결과 화면 표시 → 랭킹·기록에 누적 반영까지 자동으로 이어진다.

🎫 이 스텝의 상세 이슈 (9) →

선행: Step 3(RaceFinished 이벤트) — 정산의 트리거
BACKEND 정산 체인 (RaceFinished 구독)
ScoringModule — 서버 권위 4축(등수300·거리200·완주220·여유100, ≤1000). 배점 밸런싱은 이후, MVP는 초안 그대로
RaceResult 저장 — 유저별 완주여부·거리·완주시간·등수·점수, 멱등
RunnerStats 누적 — totalScore·거리·경기수·승률, 티어 계산(컷 기준 임시)
랭킹 — 전체 누적 + 주간/월간(RaceResult 날짜 윈도우 or Redis ZSET)
엔드포인트 GET /rankings?scope=, GET /me/results(기록), GET /races/:id/result
APP 화면 실연동
결과 화면 — 시상대·순위·점수 4축 분해 실데이터(완주/DNF)
랭킹 — 전체/주간/월간 세그먼트 실데이터, 내 랭킹 카드·티어
기록 — 지난 경주 리스트 + 상세, 프로필 누적 통계(거리·경기·승률)
mock 데이터·레이어 제거, 전 화면 실 repository로 교체
완료 기준 — 경주를 마치면 즉시 결과 화면에 점수가 뜨고, 랭킹·기록·프로필 통계에 누적 반영된다. 주간/월간 랭킹이 날짜 윈도우로 맞게 집계됨.
