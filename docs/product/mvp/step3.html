Step 3 · 경주 코어 — 실 GPS + 실시간 순위 난이도 최상

이 앱의 심장. 예약 시각 동시출발 → 각자 GPS로 뛴 거리 측정 → 좌표 없이 거리만 실시간 공유 → 순위 경쟁 → 완주/종료.

목표 — 서로 다른 장소의 참가자들이 동시에 출발해, 각자 실제로 뛴 거리가 실시간 리더보드에 반영되고 완주/시간초과로 경주가 끝난다.

🎫 이 스텝의 상세 이슈 (11) →

선행: Step 2(방·로비·스케줄러·socket.io) — 경주는 LIVE 전환된 방에서 시작
APP 실 GPS 추적 (가장 어려운 부분)
geolocator 백그라운드 추적 — iOS UIBackgroundModes[location]·allowBackgroundLocationUpdates, Android 포그라운드 서비스
거리 계산 + 노이즈 필터 — 저정확도 좌표 버림, 미세 드리프트/순간이동 무시, 누적 km 스트림(RunTracker)
좌표는 서버 미전송, 누적거리(km)만 주기 보고 — 프라이버시 원칙 유지
화면잠금·백그라운드에서도 측정 지속, 배터리 고려
BACKEND Race 도메인 & 게이트웨이
완료RaceScheduler — RoomLive 구독(RaceCreationHandler)이 Race 생성(goal=Room 목표거리/제한시간 승계, 전원 RUNNING), 서버시계 기준 동시출발(startedAt), 생성 멱등(roomId 유니크+count 선검사) S3-3
완료RaceGateway(socket.io /race) — 전용 네임스페이스, JWT 핸드셰이크, joinRoom(러너만) / progress(거리) / 3초 주기 leaderboard(거리 내림차순) 브로드캐스트 / raceFinished S3-4
완료RaceService 안티치트 — MAX_FEASIBLE_SPEED 24km/h 상한(Δt 기반 실현가능거리 초과 리젝)·역행 리젝·목표초과 클램프·Δt≤0 리젝, 리젝 시 상태 완전 불변 S3-5
완료종료 판정 — 전원 완주 자동종료 or 제한시간 지연잡(startedAt+limitMinutes) → FINISHED, 미완주 DNF, finalize 멱등(중복 호출 no-op) S3-6
완료RaceParticipant 상태(RUNNING/FINISHED/DNF/DISCONNECTED), 이벤트 RaceStarted·RunnerFinished·RaceFinished 발행
APP 경주 화면 실연동
socket.io 클라이언트 — 진행률 보고 + leaderboard 수신, 세로 리더보드 실시간 재정렬(이미 위젯 있음)
타겟 격차(Δ)·내 페이스·남은시간 실데이터, 완주/DNF 결과 진입
오프라인 내성 — 끊겨도 로컬 GPS로 내 화면 지속, 복귀 시 재동기화
DESIGN 경주 엣지 UX
연결끊김 배너·재동기화, GPS 약함/미확보, 중도 포기(DNF) 확인, 마지막 1명 처리
카운트다운/완주 햅틱·사운드 — 경쟁 손맛
주의 — 실외 실기기 필드 테스트가 필수(시뮬레이터 GPS 주입으론 한계). 2대 이상으로 서로 다른 경로를 실제로 뛰며 동시출발·순위·안티치트를 반복 검증. 시계 동기화(서버 권위) 어긋나면 공정성 붕괴 → 최우선 확인.
완료 기준 — 서로 다른 장소의 실기기 2대가 예약 시각에 동시 출발 → 각자 뛴 거리가 상대 화면 리더보드에 실시간 반영 → 목표거리 완주 또는 제한시간 종료로 경주가 정상 마감. 좌표는 서버에 전송되지 않음을 확인.
