Step 4 · 점수 · 랭킹 · 기록

경주 종료 → 4축 점수 → 랭킹·기록 누적. 보상 루프. (총 9 이슈)

📋 MVP Step 4 ← 이슈 보드
S4-1ScoringModule (4축 · 서버 권위)BACKENDM
경주 1건의 점수를 서버가 확정. 클라는 표시만.
  • 완료4축 — 등수≤300·거리≤200·완주220·여유≤100 (합계 ≤1000)
  • 완료미완주자 처리(완주보너스 0, 거리·등수 비례). 배점 밸런싱은 이후, MVP는 초안
통과 — 서버 권위 순수함수로 4축(등수≤300·거리≤200·완주220·여유≤100, 합계≤1000)을 계산하며, 미완주자는 완주보너스 0으로 거리·등수에 비례해 결정적으로 산출된다(등수는 finishTime→거리 순 산정). 동일 입력에 항상 동일 결과. tsc 0에러·jest 63 통과·부팅 검증(인증 401 확인) 완료.
선행: S3-6관련MVP도메인
바로가기📦 Score🗺️ MVP
S4-2RaceResult 저장 (RaceFinished 구독)BACKENDM
경주 종료 이벤트를 받아 유저별 결과를 원천 데이터로 저장.
  • 완료RaceFinished 구독(ScoringResultHandler) → 참가자 수만큼 유저별 완주여부·거리·완주시간·등수·점수 저장
  • 완료멱등 — (raceId, userId) 유니크 + count 선검사(중복 이벤트에도 1건), pointsAwarded 계산(적립 반영은 MVP 미노출)
통과 — RaceFinished 구독 핸들러가 참가자 수만큼 정확히 RaceResult를 저장하며, (raceId, userId) 유니크 제약 + count 선검사로 중복 이벤트 소비에도 멱등(중복 저장 없음)이 실측됐다. tsc 0에러·jest 63 통과·부팅 검증 완료.
선행: S4-1, S3-6관련MVP도메인
바로가기📦 RaceResult🗺️ MVP
S4-3RunnerStats 누적 + 티어BACKENDM
전체 누적 지표와 티어. 프로필·랭킹의 기반.
  • 완료totalScore·totalDistanceKm·raceCount·winCount 누적 갱신
  • 완료티어 계산(브론즈~다이아, 컷 기준 임시) + RaceStatApplied 마커로 멱등
통과 — 경주 종료마다 내 누적 통계(총점·거리·경기수·승수)·티어가 갱신되며, RaceStatApplied 마커로 같은 경주에 대한 중복 적용이 방지된다. S4-2(RaceResult)와 같은 RaceFinished에 반응하는 둘째 트랜잭셔널 핸들러이며, 디스패처가 핸들러별로 ALS를 격리해 두 트랜잭션이 교차오염 없이 안전함을 확인했다. tsc 0에러·jest 63 통과.
선행: S4-2관련MVP도메인
바로가기📦 RunnerStats🗺️ MVP
S4-4랭킹 (전체 / 주간 / 월간)BACKENDM
누적 + 롤링 윈도우 랭킹. 명시적 Season 테이블 없이 날짜 윈도우로.
  • 완료전체 누적(RunnerStats) + 주간/월간(RaceResult createdAt 7/30일 윈도우 SQL 집계, Season 테이블 없음)
  • 완료내 순위·주변 ±3 순위 조회
통과 — 전체(RunnerStats 누적)·주간·월간(RaceResult createdAt 기준 7/30일 SQL 날짜 윈도우 집계) 랭킹이 올바른 기간으로 산출되고, 내 순위·주변 ±3위 조회가 동작한다. Redis ZSET 기반 Leaderboard는 MVP에 미도입(후속 최적화). tsc 0에러·jest 63 통과.
선행: S4-2관련MVP도메인
바로가기📦 RunnerStats📦 Leaderboard🗺️ MVP
S4-5랭킹 / 기록 / 결과 APIBACKENDS
앱 화면이 소비할 조회 엔드포인트.
  • 완료GET /rankings?scope=all|weekly|monthly, GET /users/me/results, GET /races/:id/result — 전부 UserGuard 인증
통과 — 세 엔드포인트 모두 UserGuard로 보호된 GET이며, 목록은 { data: { items, total } }, 단건은 { data: {...} } 응답 포맷을 따른다. 결과·랭킹·기록 화면이 필요한 데이터를 한 번에 받는다. tsc 0에러·jest 63 통과·부팅+인증 401 확인.
선행: S4-2, S4-3, S4-4관련MVP도메인
바로가기📦 RaceResult📦 RunnerStats🗺️ MVP
S4-6결과 화면 실연동APPS

통과 — 경주를 마치면 실제 순위·점수 4축 분해(완주/DNF)가 결과 화면에 표시된다. flutter analyze 이슈 없음 · 테스트 133개 통과. 강건화(S5-2 연동) — 백엔드가 리더보드 스냅샷에 raceId를 노출해 앱이 종료 후 GET /races/:id/result를 직접 조회하도록 바꿔, 반환 유저가 이전 경주 결과를 보게 되는 위험을 없앴다(최신기록 폴링 우회 제거).
