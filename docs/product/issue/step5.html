Step 5 · 알림 & 출시 준비

예약 경주 푸시 + 스토어 심사에 필요한 에셋·정책·배포까지. (총 9 이슈)

📋 MVP Step 5 ← 이슈 보드
S5-1FCM/APNs + DeviceToken + Notification 발송BACKENDM
예약 경주 모델의 필수 요소. 이벤트를 구독해 푸시 발송.
  • 완료POST/DELETE /api/users/me/device-tokens(token UNIQUE upsert·소유자 재귀속) + FirebasePushSender(크레덴셜 optional — 미설정 시 초기화 스킵+no-op, 유효 시 sendEachForMulticast 실발송)
  • 완료이벤트 구독(팬아웃 핸들러) — ParticipantJoined→방장, RoomStarting(T-10s)→참가자 전원, RaceFinished→러너 전원. dedupeKey(type:room:user) 유니크로 멱등(신규 대상만 발송), 발송 실패 비치명
  • 완료선택 T-1분 리마인더 지연잡 배선 — 트리거가 RoomStarting(T-10s)이라 실질은 dedupeKey 재사용 즉시·중복없음 안전망(진짜 1분전 예고 아님, 정직하게 표기)
완료 — DeviceToken 등록/삭제, 이벤트 구독 팬아웃 발송(3종), dedupeKey 멱등까지 code-reviewer 검증 통과(tsc 0에러·jest 81 통과·크레덴셜 없이 부팅 성공·핸들러 8개 확인). 미검증: 실 FCM 발송·실기기 도달은 크레덴셜+실기기 필요(코드/배선까지만 확인).
선행: S4-2관련MVP도메인
바로가기📦 DeviceToken📦 Notification🗺️ MVP
S5-2푸시 수신 / 권한 / 딥링크APPM
앱에서 알림 권한을 받고 수신·탭 시 올바른 화면으로 이동.
  • 완료firebase_messaging 알림 권한 요청(iOS·Android 13+ POST_NOTIFICATIONS) + FCM 토큰 획득 → 로그인/앱 시작 시 POST /users/me/device-tokens{token,platform} 등록, onTokenRefresh 재등록, 로그아웃 시 DELETE(백엔드 계약 {token} 일치)
  • 완료포그라운드 수신(인앱 배너/스낵바), 백그라운드 탭(onMessageOpenedApp)·콜드스타트(getInitialMessage) 딥링크 — RACE_STARTING/PARTICIPANT_JOINED→해당 방 로비, RESULT_READY→결과 화면(raceId)
  • 완료Firebase 미설정 시 graceful(initializeApp try/catch → 푸시만 비활성, 백엔드 no-op와 대칭)
완료 — 알림 권한 요청·토큰 등록/재등록/로그아웃 삭제, 포그라운드 수신, 백그라운드 탭·콜드스타트 딥링크(방 로비/결과 화면 raceId)까지 flutter analyze 이슈 없음 · 테스트 167개 통과(fake 기반)로 검증됐다. 미검증: 실 FCM 수신·백그라운드/콜드스타트 딥링크·APNs는 실기기 + 사용자 Firebase 프로젝트 파일(google-services.json/GoogleService-Info.plist) 배치 후에만 검증 가능(현재 미배치 — PUSH_SETUP.md에 절차 안내). Android google-services gradle 플러그인은 파일 부재로 미적용 상태.
선행: S5-1관련MVP화면
바로가기🎨 로비🎨 결과📦 Notification🗺️ MVP
