Step 3 · 경주 코어 — 실 GPS + 실시간 순위

이 앱의 심장. 동시출발 → 각자 GPS 거리측정 → 좌표 없이 실시간 순위 → 완주/종료. (총 11 이슈, 난이도 최상)

📋 MVP Step 3 ← 이슈 보드
S3-1RunTracker: 백그라운드 GPS + 노이즈필터APPL
실제로 뛴 거리를 정확히 측정하는 핵심 모듈. 좌표가 아니라 누적 거리만 산출.
  • geolocator 위치 스트림, 화면잠금·백그라운드에서도 지속
  • 노이즈필터 — 저정확도(accuracy 큰) 버림, 미세 드리프트/순간이동 점프 무시
  • 누적 거리(km) 스트림 노출, 배터리 고려
완료 — 실제 이동 시 누적 거리가 합리적으로 증가하고 정지 시 드리프트가 누적되지 않는다.
선행: S3-2관련MVP화면
S3-2위치 권한 (항상 허용/백그라운드)APPM
경주 중 화면을 꺼도 측정하려면 백그라운드 위치 권한이 필요.
  • iOS: UIBackgroundModes[location]·allowBackgroundLocationUpdates·Info.plist 문구
  • Android: 위치·FOREGROUND_SERVICE_LOCATION·포그라운드 서비스 알림
  • 권한 거부 플로우(설정 유도) 연결
완료 — 항상 허용 권한 취득 후 백그라운드에서도 위치 업데이트가 들어온다.
선행: S1-11관련MVP화면
S3-3RaceScheduler: Race 생성 · 동시출발BACKENDM
방이 LIVE로 전환되면 Race를 만들고 서버시계 기준 동시출발 시각을 확정.
  • 완료RoomLive 이벤트 구독(RaceCreationHandler) → Race 생성 — goal은 Room의 targetDistance·limitMinutes 승계, 러너 전원 RUNNING
  • 완료startedAt 서버시계 now로 전원 동일하게 확정, RaceStarted 이벤트 발행
  • 완료생성 멱등 — roomId 유니크 제약 + count 선검사(중복 RoomLive 소비에도 Race 중복 생성 없음)
통과 — 방이 LIVE로 전환되면 Race가 생성되고(goal=Room 목표거리/제한시간 승계, 참가자 전원 RUNNING), 서버시계 기준 동일한 startedAt이 전원에게 전달된다. roomId 유니크 제약 + count 선검사로 생성이 멱등(중복 생성 없음). domain-events 큐 단일 워커의 팬아웃 디스패처(DiscoveryService 핸들러 수집 → supports 매칭, 핸들러별 격리 ALS 컨텍스트)가 같은 RoomLive 이벤트를 로비 브로드캐스트(S2-4)와 Race 생성 두 핸들러에 동시 전달함을 확인. tsc 0에러·jest 42 통과·실부팅 성공.
선행: S2-3관련MVP도메인
바로가기📦 Race🗺️ MVP
S3-4RaceGateway (/race socket.io)BACKENDL
경주의 실시간 중계. 거리만 받아 순위를 브로드캐스트(좌표 미전송).
  • 완료/race 전용 네임스페이스, JWT 핸드셰이크(미인증 소켓 거부)
  • 완료joinRoom(러너만 허용) · progress(거리만, 좌표 미전송) · 3초 주기 leaderboard(거리 내림차순) 브로드캐스트 · raceFinished
  • 완료네임스페이스 전용 RaceBroadcaster — 로비(기본 네임스페이스, S2-4)와 완전 분리
통과 — /race 네임스페이스가 JWT 핸드셰이크로 미인증 접속을 거부하고, 러너만 joinRoom할 수 있으며, 거리 progress 보고가 3초 주기 leaderboard(거리 내림차순) 브로드캐스트로 같은 방 전원에게 실시간 반영된다. 로비와 네임스페이스가 분리돼 있어 상호 간섭이 없다. tsc 0에러·jest 42 통과·실부팅 성공.
선행: S2-4, S3-3관련MVP도메인
바로가기🎨 경주📦 Race📦 RaceParticipant🗺️ MVP
S3-5RaceService 안티치트 · 진행 클램프BACKENDM
공정성의 핵심. 비현실적 진행은 서버가 최종 판정해 거부.
  • 완료MAX_FEASIBLE_SPEED 24km/h 상한 — 보고 간격(Δt) 기준 실현가능거리를 초과하면 리젝(순간이동/비현실 페이스 방지)
  • 완료역행(거리 감소) 리젝, 목표거리 초과분 클램프
  • 완료Δt ≤ 0(비정상 타임스탬프) 리젝 — 리젝 시 참가자 상태 완전 불변(값 변경 없음)
통과 — 직전 보고 이후 경과시간(Δt) 기준 MAX_FEASIBLE_SPEED(24km/h)로 계산한 실현가능거리를 초과하는 보고, 역행 보고, Δt≤0 보고가 모두 리젝되고 리젝 시 상태가 완전히 불변으로 유지된다. 목표거리 초과분은 클램프돼 순위에 과대반영되지 않는다. tsc 0에러·jest 42 통과·실부팅 성공.
선행: S3-4관련MVP도메인
바로가기📦 RaceParticipant🗺️ MVP
S3-6종료 판정 + finalize 멱등BACKENDM
경주를 안전하게 마감. 중복 종료 호출에도 결과가 한 번만 확정.
  • 완료전원 완주 시 자동 종료, 또는 제한시간 지연잡(startedAt + limitMinutes)으로 FINISHED 전환
  • 완료미완주 러너는 DNF 처리, RunnerFinished·RaceFinished 이벤트 발행
  • 완료finalize 멱등 — 이미 FINISHED면 no-op(중복 RaceFinished 발행 없음)
통과 — 전원 완주 시 즉시, 미완주자가 있으면 제한시간 지연잡 도달 시 Race가 FINISHED로 전환되고 미완주 러너는 DNF로 확정된다. RunnerFinished·RaceFinished 이벤트가 발행되며, finalize를 중복 호출해도 이미 FINISHED면 no-op(중복 이벤트·결과 변경 없음)이 실측됐다. tsc 0에러·jest 42 통과·실부팅 성공.
선행: S3-4, S3-5관련MVP도메인
바로가기📦 Race🗺️ MVP
S3-7경주 화면 socket 실연동APPM
RunTracker 거리를 보고하고 서버 리더보드를 받아 실시간 순위 표시.
  • socket.io 클라 — 거리 progress 전송 + leaderboard 수신
  • 세로 리더보드 재정렬(위젯 존재)·타겟 격차(Δ)·페이스·남은시간 실데이터
  • 완주/DNF 결과 화면 진입
완료 — 내가 뛴 거리가 상대 화면에, 상대 거리가 내 화면에 실시간으로 반영된다.
선행: S3-1, S3-4관련MVP화면
S3-8오프라인 내성APPM
경주 중 네트워크가 끊겨도 내 측정은 지속되고 복귀 시 재동기화.
  • 끊김 시 로컬 GPS로 내 화면 계속, 거리 버퍼링
  • 복귀 시 재전송·재동기화, 연결끊김 표시
완료 — 일시적 끊김 후 복귀하면 그 사이 거리가 반영되어 순위가 맞춰진다.
선행: S3-7관련MVP화면
S3-9경주 엣지 UXDESIGNM
경주 중 발생하는 예외 상황들의 화면·안내.

  • 중도 포기(DNF) 확인, 마지막 1명 남음 처리
완료 — 각 예외 상황에서 사용자가 상태를 이해하고 대응할 수 있다.
