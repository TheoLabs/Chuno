# 추노(Chuno) 모바일 앱 — 설계 & 디자인 문서

> 상태: MVP 설계 (2026-07-01). 코드 구현 전 기획/디자인 단계.
> 범위: `apps/chuno-mobile` (Flutter, iOS/Android). 백엔드 계약은 참고용으로만 표기.

---

## 0. 한 줄 정의

**장소에 구애받지 않고, 각자 GPS로 뛴 거리를 실시간 공유하며 경쟁하는 러닝 앱.**
혼자 뛰는 게 아니라 "런닝방"에 모여 같은 목표(거리·제한시간)를 두고 동시에 출발해 순위를 겨룬다.

---

## 1. 확정된 핵심 규칙 (2026-07-01 결정)

| 항목 | 결정 |
| --- | --- |
| **승리 조건** | 제한시간 내 **목표거리 완주 경쟁**. 완주자는 완주 시각 순으로 등수, 미완주자는 달린 거리 순으로 그 뒤. |
| **"목표 시간대" 의미** | = 완주 **제한시간**(예: 40분). 시작시각과는 별개. |
| **경주 시작** | 방 생성 시 정한 **예약 시각에 서버시계 기준 전원 동시 출발**. → 푸시 알림 필수. |
| **방 찾기** | **공개 방 목록 + 필터**(목표거리/제한시간/인원). MVP 매칭 대체. |
| **경주 화면** | **세로 리더보드**(순위 행 + 진행률 바, 실시간 갱신). |
| **위치 프라이버시** | 좌표는 서버 미전송. **누적 거리(km)만** 공유. |
| **최소 인원** | **2명.** 시작 시각에 2명 미만이면 **방 자동 삭제**(경주 미성립). *(2026-07-01 확정)* |

## 2. 화면 목록 (IA)

```
Auth
 ├─ 로그인 (카카오 / 구글 / 애플)
 └─ 온보딩 (닉네임 → 러닝레벨 → 위치권한 → 완료)   ※ 최초 1회

MainShell (하단 탭 4개)
 ├─ [홈]     공개 방 목록 + 필터  →  방 상세/로비
 │            └─ 방 생성
 ├─ [랭킹]   전체 / 주간 / 월간 리더보드 + 티어
 ├─ [기록]   내 지난 경주 히스토리  →  경주 상세
 └─ [프로필] 내 정보 · 포인트 · 설정
              ├─ 프로필 수정
              ├─ 포인트/스토어 (광고제거 한달권 등)
              └─ 설정 (알림/권한/로그아웃)

경주 플로우 (풀스크린, 탭 밖)
 로비(대기) → 시작대기(카운트다운) → 경주(세로 리더보드) → 결과(시상대)
```

MVP 총 화면 ≈ 14개.

## 3. 상태머신

### 방(Room) 상태
```
RECRUITING(모집중) ──예약시각 T-Δ──▶ STARTING(카운트다운)
     │  방장 삭제 / 시작시각 인원미달                     │  카운트다운 종료
     ▼                                                   ▼
 CANCELLED(취소)                                    LIVE(진행중)
                                                         │  전원 완주 or 제한시간 종료
                                                         ▼
                                                    FINISHED(종료)
```

### 참가자(Participant) 상태 (LIVE 중)
```
READY ──출발──▶ RUNNING ──목표거리 도달──▶ FINISHED(완주)
                   │                          
                   ├─ 제한시간 종료(미도달) ─▶ DNF(미완주, 거리 기록됨)
                   ├─ 사용자 포기         ─▶ DNF
                   └─ 연결끊김            ─▶ DISCONNECTED (로컬 GPS는 계속 측정, 복귀 시 재동기화)
```

## 4. 화면별 설계

### 4.1 로그인
- 다크 배경 + 브랜드 로고/태그라인("혼자 뛰지 마라, 겨뤄라").
- 소셜 버튼 3개: **카카오**(노랑) · **구글**(4색 G) · **애플**(검정, iOS 필수 정책).
- 하단: 이용약관/개인정보 링크.

### 4.2 온보딩 (최초 1회, 4스텝 PageView)
1. **닉네임** — 중복확인, 2~12자.
2. **러닝 레벨** — 입문 / 중급 / 고급 (방 추천·매칭 힌트용).
3. **위치 권한** — *앱 핵심 컨셉* 설명 후 "항상 허용/백그라운드" 요청. 거부 시 "경주 참여 불가" 안내 + 설정 유도.
4. **완료** — MainShell로 진입.

### 4.3 홈 — 공개 방 목록 + 필터
```
┌──────────────────────────────┐
│  추노            🔔   ⚙︎        │
│  ┌ 필터 ─────────────────────┐ │
│  │ 거리▾  제한시간▾  인원▾     │ │
│  └──────────────────────────┘ │
│  🔴 LIVE 배너 (진행중 방)       │
│ ┌──────────────────────────┐ │
│ │ 🏃 5km 새벽 러닝          │ │
│ │ 목표 5.0km · 40분 · 3/6명 │ │
│ │ ⏰ 06:00 시작 (12분 후)    │ │
│ │           [ 참가 ]         │ │
│ └──────────────────────────┘ │
│ ┌──────────────────────────┐ │
│ │ 🏃 10km 챌린지  · 2/8명   │ │
│ │ ⏰ 07:30 시작              │ │
│ └──────────────────────────┘ │
│                       ( + )   │  ← 방 생성 FAB
└──────────────────────────────┘
```
- 카드 정보: 방이름, 목표거리·제한시간·현재/최대 인원, 시작까지 카운트다운, 상태 뱃지(모집중/곧시작/LIVE).
- 필터: 거리대(3/5/10km…), 제한시간, 남은 자리, 시작 임박순 정렬.
- 빈 상태: "아직 방이 없어요 — 첫 방을 만들어보세요".

### 4.4 방 생성
입력 항목:
- 방 이름
- **목표 거리(km)** — 프리셋(3/5/10) + 커스텀
- **제한 시간(분)**
- **최대 참가 인원**
- **시작 시각** — 날짜/시간 피커 (지금+최소 N분 뒤부터)
- (선택) 공개/비공개 — MVP는 공개 기본
- 하단 요약 미리보기 → [방 만들기]

### 4.5 방 로비 (대기실)
```
┌──────────────────────────────┐
│ ← 5km 새벽 러닝                │
│  목표 5.0km · 제한 40분        │
│  ⏰ 06:00 시작  (남은 12:03)    │
│ ──────────────────────────── │
│  참가자 3/6                    │
│  🟢 나(방장)   🔵 준호  🟣 민지 │
│  ⬚ 빈자리  ⬚ 빈자리  ⬚ 빈자리  │
│ ──────────────────────────── │
│  ⓘ 시작 시각에 자동 출발합니다   │
│                               │
│  [ 나가기 ]    (방장:[방 삭제]) │
└──────────────────────────────┘
```
- 실시간 참가자 입퇴장 반영(WS).
- 카운트다운이 T-10초 되면 자동으로 **시작대기(카운트다운) 화면**으로 전환.
- 방장만 방 삭제 가능(모집중 한정).

### 4.6 시작대기 (카운트다운)
- 풀스크린 3·2·1 카운트다운 + 진동/사운드. 서버시계 동기화 기준.
- "위치 확보 중" 상태 표시(GPS fix 확인). 실패 시 경고.

### 4.7 ★ 경주 화면 (세로 리더보드) — 앱의 핵심
```
┌──────────────────────────────┐
│ 🏁 목표 5.0km · 남은시간 18:42  │  ← 상단 고정 헤더
│ ──────────────────────────── │
│ 1  🟢 나      3.2km ▓▓▓▓▓░ 64% │  ← 내 행 하이라이트(시안)
│ 2  🔵 준호    3.0km ▓▓▓▓░░ 60% │
│ 3  🟣 민지    2.7km ▓▓▓░░░ 54% │
│ 4  🟠 서연    2.1km ▓▓░░░░ 42% │
│ ──────────────────────────── │
│  내 페이스 5'30"/km            │  ← 내 상세 스트립
│  남은거리 1.8km · 예상 완주 9분 │
│                               │
│         [ 그만두기 ]           │
└──────────────────────────────┘
```
- **행 = 참가자**: 순위번호, 컬러 아바타, 닉네임, 누적거리, 진행률 바(목표 대비 %).
- **실시간**: 2~5초마다 순위 재정렬(부드러운 리스트 애니메이션). 내 행은 항상 강조.
- 완주자는 상단에 🏁 뱃지 + "완주 3위" 고정, 아래로 진행 중 러너.
- 하단 내 스트립: 페이스, 남은거리, 예상 완주 시간, 앞사람과의 격차.
- **오프라인 내성**: 네트워크 끊겨도 로컬 GPS로 내 거리/화면은 계속 갱신, 남 순위만 "동기화 중" 흐림 처리.
- 완주 순간: 풀스크린 축하 연출 → 결과 대기(남은 사람 기다림) 또는 바로 결과.

### 4.8 결과 (시상대)
```
        🥈        🥇        🥉
       준호        나        민지
      완주 12'40  완주 12'02  DNF 4.6km
 ───────────────────────────────
  내 결과: 1위 · 완주 5.0km · 12'02
  획득 점수 820  ·  +82 P
  [점수 상세 ▾]  등수 300 / 거리 200
                완주 220 / 여유 100
 ───────────────────────────────
   [ 다시하기 ]        [ 홈으로 ]
```
- 시상대(1~3위) + 전체 순위 리스트 + 내 결과 카드.
- **점수 분해** 상세시트(등수/거리/완주/여유).
- 획득 포인트 표시. (다시하기 = 같은 조건 방 재생성 → v2 고려)

### 4.9 랭킹
- 세그먼트: **전체 누적 / 주간 / 월간**.
- 내 랭킹 카드(상단 고정) + 리더보드 리스트 + **티어**(브론즈~다이아) 뱃지.

### 4.10 기록(히스토리)
- 지난 경주 리스트(날짜·방·등수·거리·점수) → 탭 시 경주 상세(당시 리더보드/기록).

### 4.11 프로필 / 포인트 / 설정
- 프로필: 아바타, 닉네임, 레벨, 티어, 누적거리·경기수·승률, 보유 포인트.
- **포인트/스토어**: 광고제거 한달권 등 교환. (MVP는 항목 1~2개)
- 설정: 알림 on/off, 위치권한 재요청, 로그아웃/탈퇴.

## 5. 점수 & 포인트 설계 (초안 — 배점/밸런싱은 MVP 이후)

> **결정(2026-07-01): 점수 배점·포인트 적립/가격 밸런싱은 MVP 이후로 연기.** MVP는 아래 초안 공식을 그대로 사용하고(동작 확인 목적), 실제 수치 튜닝은 출시 후 데이터 보고 진행.

경주 1건당 **Chuno Score (최대 1000)** = 4축 가중합. 승리조건(제한시간 내 완주)에 맞춰 완주 비중을 크게:

| 축 | 배점 | 산식(개념) |
| --- | --- | --- |
| 완주 보너스 | 220 | 목표거리 완주 시 만점, 미완주 0 |
| 등수 | 300 | 참가인원 대비 등수(1위 만점 → 꼴찌 하한) |
| 거리 | 200 | min(달린거리/목표거리, 1) 비례 (미완주자도 뛴 만큼) |
| 완주 여유 | 100 | 완주자 한정, 제한시간 대비 여유(빨리 완주할수록↑) |
| **(선택) 페이스** | 180 | 일정 이상 페이스 유지 보너스 — *이전 4축 설계 참고, 채택 여부 결정* |

> 배점 합/세부 계수는 **밸런싱 필요**. 서버 권위로 계산(클라는 표시만).

**포인트 지급(초안)**: `획득 P = round(Score / 10)` (예: 820점 → 82P) + 완주 시 고정 보너스.
- DNF·꼴찌도 소량 참가 보상(리텐션) — 예: 최소 10P.

**포인트 소비(MVP)**: 광고제거 한달권 (예: **500P**). 참고로 완주 1회 ≈ 50~90P → 약 6~10회 완주로 1개월권.
> 적립률 ↔ 가격은 함께 튜닝(포인트 인플레 방지).

## 6. 디자인 시스템 — "추격전 HUD" (2026-07-01 확정)

> **방향 결정**: 초기 "다크+네온 라임" 안은 Nike/Strava류 피트니스 앱과 구분이 안 되는 흔한 템플릿이라 폐기. 대신 이름 추노(推奴=*쫓다·추격*)의 본질을 정면으로 살린 **추격전 HUD** 정체성 채택. 핵심 감정 = 성장/격려가 아니라 **긴장·추격·라이벌**. 앞사람=**타겟(락온)**, 뒷사람=**추격자**, 거리보다 **격차(Δ)**가 주인공.

- **테마**: 잉크블랙 다크. 배경 `#0B0C10`, 패널 `#151821`/`#1C2029`, 라인 `#272C36`.
- **컬러 (2026-07-01 확정 — 선셋 코랄)**: **메인 선셋 코랄 `#FF6B4A`**(primary·나), 그 위 텍스트 `#210a04`, **강조/타겟 핑크레드 `#FF3D77`**(앞사람=타겟), 완주 `#37D67A`, 텍스트 `#ECEFF4`, 뮤트 `#8A929D`. *메인 색은 코랄로 확정하되 프로토타입에 앰버/바이올렛/민트/블루 스위처는 참고용으로 남겨둠.* 글로우·틴트는 메인 색에서 `color-mix`로 파생.
- **타이포**: 숫자·시계·거리 등 데이터만 **모노스페이스**, 제목·라벨·본문은 산세리프(대문자/과한 자간 지양). 부드러운 톤.
- **모티프**: 은은한 **앰버(코랄) 글로우 + 얇은 좌측 액센트**(각진 브래킷 대신), **타겟 강조/격차 칩(Δ)**, 소프트 레티클 카운트다운, 둥근 코너(18px)·필 칩/태그·원형 아바타.
- **핵심 컴포넌트**: MissionBar(목표·남은시간), RaceRow(격차 칩·range 바), LockStrip(타겟까지 Δ), RoomCard, CountdownReticle, Podium, TierBadge, ScoreBreakdown, SocialLoginButton.
- **모션(시그니처)**: 경주 리더보드 **순위 재정렬**(translateY transition), 카운트다운 레티클, 완주 연출. → **HTML 프로토타입 `docs/wireframes/index.html`에 실제 구현**(거리 증가→재정렬 애니메이션 + 클릭 플로우 + 상태 화면들).

## 7. 모바일 기술 관점 처리 (설계 메모)

- **실시간**: socket.io 클라이언트. `joinRoom` → 서버가 주기 `leaderboard` 브로드캐스트. 내 진행은 2~5초마다 `progress`(거리만) 전송.
- **GPS**: `geolocator` 백그라운드 추적. 노이즈 필터(저정확도 좌표 버림, 미세 드리프트/순간이동 무시). 화면잠금/백그라운드에서도 측정(iOS UIBackgroundModes[location], Android foreground service).
- **시계 동기화**: 동시출발이 생명 → 서버 시각 기준 오프셋 계산(`elapsed = now - scheduledStartAt`).
- **오프라인 내성**: 로컬 거리 버퍼링 후 복귀 시 재전송/재동기화.
- **안티치트(클라 측 협조)**: 비현실 페이스/점프는 서버가 최종 판정, 클라는 필터링만.
