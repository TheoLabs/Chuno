추노 MVP 로드맵

"장소 무관 실시간 경쟁 러닝"의 최소 기능. 설계·UI 껍데기(mock)는 완료됨 → 이제 핵심 엔진(백엔드·실시간·GPS)을 스텝으로 쌓는다.

🎫 상세 이슈 보드 (스텝별 티켓) →

MVP 핵심 루프 — 이 한 바퀴가 돌면 MVP다. 나머지는 전부 이후로.
로그인→ 방 생성·참가→ 예약 동시출발→ 실 GPS 거리 실시간 공유→ 순위·점수→ 랭킹·기록

포함 / 제외 scope

MVP 포함

  • 소셜 로그인(카카오·구글·애플) + 온보딩
  • 공개 방: 생성·목록·참가·나가기, 인원미달 자동취소
  • 예약 시각 동시출발(서버시계)
  • 실 GPS 거리측정 + socket.io 실시간 순위(좌표 미전송)
  • 기본 안티치트(비현실 페이스 리젝)
  • 점수(4축)·완주판정·랭킹(전체/주간/월간)·기록
  • 경주 임박·결과 푸시 알림

MVP 제외 (이후)

  • 포인트 상점 / 광고 제거(적립·밸런싱 미정)
  • Kafka EDA → MVP는 인프로세스 이벤트 + BullMQ로
  • 프로필 사진 업로드(S3)
  • 비공개 방(초대코드)·자동 매칭
  • 리매치, 티어 정교화, 지도/경로 공유
  • 백오피스(back-office)

APPBACKENDDESIGN

APPOPSDESIGN

주요 리스크

핵심 리스크는 Step 3에 몰려 있다. 실 GPS 정확도·백그라운드 추적, 실시간 동기화(서버시계 동시출발), 공정성(안티치트)이 이 앱의 성패. → Step 1~2로 로그인/방을 빨리 세우고, Step 3을 최우선·최다시간 배정. 소셜로그인 3종·푸시는 각각 개발자계정/키 설정 리드타임이 있으니 미리 발급 착수.
