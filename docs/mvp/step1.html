MVP 로드맵/Step 1

Step 1 · 기반 & 인증

백엔드 골격을 세우고, 소셜 로그인 → 온보딩 → 홈 진입까지 실계정으로 도는 첫 뼈대.

목표 — 카카오/구글/애플로 로그인해서 닉네임·레벨을 저장하고 홈에 들어간다. (아직 방·경주는 mock이어도 됨)

🎫 이 스텝의 상세 이슈 (12) →

선행: 없음 (시작점)
BACKEND core-api 골격 & Auth
완료ConfigModule(@nestjs/config, 환경파일+validate) + TypeORM forRootAsync 연결(MySQL, charset utf8mb4) 로컬 도커 MySQL, synchronize:true — false 전환은 S1-3에서
완료헬스체크 GET /api/health 200 실측 + graceful shutdown(enableShutdownHooks · datasource.destroy()) DB ping은 스코프 제외로 확정
완료DDD 공용 기본형(libs/ddd) — AggregateRoot·DomainEvent 기본형 + 리포지토리/서비스 포트 골격 공용 DDD 기본형 골격 완료(@libs/ddd)
User / AuthIdentity 엔티티 + 마이그레이션 Nickname·RunnerLevel·Tier
UserConsent 엔티티 — 약관·개인정보·위치 동의 기록(type·documentVersion·agreedAt·revokedAt?) 법적 입증용, User와 1:N
소셜 검증 — google-auth-library / apple(jose) / kakao API로 idToken 검증, provider별
JWT 액세스(15m) + opaque 리프레시 회전, 재사용 감지 시 401
엔드포인트 POST /auth/social-login·refresh·logout, GET·PATCH /users/me, GET /users/check-nickname
AUTH_DEV_MODE — 키 없이 테스트할 dev 토큰 경로(카카오/애플 실키 발급 전 개발용)
APP Flutter 실연동 기반
완료상태관리·네트워킹 도입 — Riverpod 확정(ProviderScope) + dio + flutter_secure_storage 실제 보안 저장 S1-7, code-reviewer 실측: flutter analyze 무이슈 + 33/33 테스트 통과
완료api_client — 401 시 refresh 회전 + single-flight + 재시도 후 요청 재개(TokenStore·Env·AppException·ApiResponse 포함) 실 E2E 왕복은 백엔드 S1-5 확정 후 잔여 확인
session/AuthGate(로그인 여부 분기)
소셜 로그인 SDK — google_sign_in / sign_in_with_apple / kakao (iOS 네이티브 클라이언트 주의)
로그인 화면 실동작, 온보딩 저장(닉네임 중복확인·레벨 → PATCH /users/me)
권한 요청 플로우(위치) 연결 — 실제 권한 요청은 Step 3에서 사용
DESIGN 법적/동의
완료온보딩 약관 동의 단계 + 약관/개인정보/위치 문서 뷰어 화면(앱 내 열림) mock 구현(로컬 setState), code-reviewer 실측: flutter analyze 무이슈 + 33/33 테스트 통과
남음이용약관·개인정보처리방침 실제 문안 — 현재 placeholder(시행일 2026-00-00), 법무 검토 필요 + 개인정보처리방침 별도 문서화 검토
완료위치정보 이용동의(위치정보법 필수) 온보딩 동의 단계 반영(mock) 동의 서버 기록(UserConsent: type·documentVersion·agreedAt)은 S1-3으로 이월
소셜 개발자 콘솔 앱 등록(카카오·구글 GCP·Apple) — 키/클라이언트 ID 발급 리드타임 있음, 미리
완료 기준 — 시뮬레이터에서 실제 소셜 계정(또는 dev 토큰)으로 로그인 → 온보딩(닉네임/레벨/동의) → 홈 진입. 앱 재실행 시 세션 유지, 로그아웃 동작.
