Step 2 · 방 & 실시간 로비

방을 만들고 여러 명이 모여 예약 시각을 기다리는 흐름. 실시간(socket.io) 입퇴장이 처음 등장.

목표 — 실데이터로 방을 만들고 다른 사람이 참가해 로비에서 실시간으로 인원이 갱신되며, 예약 시각 대기 상태까지 간다.

🎫 이 스텝의 상세 이슈 (10) →

선행: Step 1(인증·User) — 방장/참가자 식별에 필요
BACKEND Room 도메인
Room 애그리거트(RaceGoal·RoomStatus·participants) + Participant 엔티티, 마이그레이션
엔드포인트 POST /rooms, GET /rooms(필터), POST /rooms/:id/join·leave, DELETE /rooms/:id(방장)
불변식 — 정원 초과 join 불가, 시작 후 join 불가, 최소 2명
예약 스케줄러(BullMQ 지연잡 or @Interval) — scheduledStartAt 도달 시 RECRUITING→STARTING→LIVE 전환, 2명 미만이면 CANCELLED
socket.io 게이트웨이(로비) — 방 룸 조인, 입퇴장·상태전환 브로드캐스트
도메인 이벤트(인프로세스) — RoomCreated·ParticipantJoined·RoomStarting·RoomCancelled
APP 방 화면 실연동
홈 방 목록 실데이터 + 필터(거리/제한시간/빈자리) + 시작 카운트다운
방 생성 폼 → POST /rooms, 성공 시 로비 이동
로비 — socket.io 연결, 참가자 실시간 입퇴장, 나가기/방삭제
서버시계 동기 카운트다운(elapsed = now − scheduledStartAt), STARTING 진입 시 카운트다운 화면 전환
엣지 상태 UI — 방 취소(인원미달) 안내, 목록 빈 상태
완료 기준 — 기기 2대(또는 시뮬 2개)에서 한쪽이 방 생성 → 다른쪽이 목록에서 참가 → 양쪽 로비에 서로 실시간 표시 → 예약 시각에 카운트다운 진입. 인원 미달 시 자동 취소 확인.
