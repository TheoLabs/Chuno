MVP 로드맵/Step 2

Step 2 · 방 & 실시간 로비

방을 만들고 여러 명이 모여 예약 시각을 기다리는 흐름. 실시간(socket.io) 입퇴장이 처음 등장.

목표 — 실데이터로 방을 만들고 다른 사람이 참가해 로비에서 실시간으로 인원이 갱신되며, 예약 시각 대기 상태까지 간다.

🎫 이 스텝의 상세 이슈 (12) →

선행: Step 1(인증·User) — 방장/참가자 식별에 필요
BACKEND Room 도메인
Room 애그리거트(RaceGoal·RoomStatus·participants) + Participant(joinedOn) 엔티티, TypeORM 스키마 S2-1
엔드포인트 POST /rooms(scheduledStartOn 미래시각만), GET /rooms(거리·제한시간·빈자리 필터, 임박순), POST /rooms/:id/join·leave, DELETE /rooms/:id(방장·모집중) S2-2
불변식 — 정원 초과·시작 후 join(늦참)·중복 join 불가. 방장 leave = 방 취소 S2-2
예약 스케줄러(BullMQ 지연잡) — scheduledStartOn 도달 시 RECRUITING→STARTING→LIVE 전환, 2명 미만이면 CANCELLED(soft), 멱등 처리 S2-3
socket.io 게이트웨이(로비) — JWT 핸드셰이크, 방 룸 조인, 입퇴장·상태전환 브로드캐스트 + 서버시각(serverTime) 제공 S2-4
도메인 이벤트(인프로세스) — RoomCreated·ParticipantJoined·RoomStarting·RoomCancelled → 로비 게이트웨이 구독 S2-5
APP 방 화면 실연동
홈 방 목록 실데이터 + 필터(거리/제한시간/빈자리) + 상태뱃지·임박순 + 빈 목록 분기 S2-6
방 생성 폼 → POST /rooms, 성공 시 로비 이동 S2-7
로비 — socket.io 연결, 참가자 실시간 입퇴장, 나가기/방삭제, 재접속 S2-8
서버시계 동기 카운트다운(오프셋 보정 remaining = scheduledStartOn − (localNow+offset)), STARTING 진입 시 카운트다운 화면 전환 S2-9
홈 방카드 "참가" → join → 로비 진입, 실패(정원초과·늦참·취소) 안내 S2-11
DESIGN 엣지 상태
방 엣지 상태 UX — 인원미달 자동취소 안내(재모집 유도), 목록 빈 상태, 방장 이탈·늦참 거부 카피 S2-10
OPS 인프라
Redis 연결·설정 + BullMQ 큐 모듈(지연잡용)·헬스체크·graceful shutdown S2-12
완료 기준 — 기기 2대(또는 시뮬 2개)에서 한쪽이 방 생성 → 다른쪽이 목록에서 참가 → 양쪽 로비에 서로 실시간 표시 → 예약 시각에 서버시계 기준 카운트다운 진입. 인원 미달 시 자동 취소 확인.
