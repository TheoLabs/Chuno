Step 5 · 알림 & 출시 준비

예약 경주 푸시 + 스토어 심사에 필요한 에셋·정책·배포까지. (총 9 이슈)

📋 MVP Step 5 ← 이슈 보드
S5-1FCM/APNs + DeviceToken + Notification 발송BACKENDM
예약 경주 모델의 필수 요소. 이벤트를 구독해 푸시 발송.
  • DeviceToken 등록 API, FCM/APNs 발송기
  • 이벤트 구독(Notification 컨텍스트) → 경주 임박(T-10/1분)·새 참가자·결과 도착
완료 — 경주 임박·결과 시 실제 기기에 푸시가 도착한다.
선행: S4-2관련MVP도메인
S5-2푸시 수신 / 권한 / 딥링크APPM
앱에서 알림 권한을 받고 수신·탭 시 올바른 화면으로 이동.
  • 알림 권한 요청, 토큰 등록, 포그라운드/백그라운드 수신
  • 딥링크 — 해당 방/결과 화면으로 이동
완료 — 푸시를 탭하면 관련 방/결과 화면으로 정확히 진입한다.
선행: S5-1관련MVP화면
S5-3앱 아이콘 + 스플래시DESIGNM
브랜드 첫인상. 추격전 정체성을 담은 아이콘·스플래시.
  • iOS/Android 전 사이즈 앱 아이콘
  • 스플래시(다크+코랄), 로고 마크
완료 — 홈 화면 아이콘·부팅 스플래시가 브랜드에 맞게 표시된다.
선행: —관련MVP
S5-4스토어 에셋DESIGNM
스토어 등록·전환율을 위한 시각·문안 자산.
  • 스크린샷·프리뷰, 앱 이름/설명/키워드
완료 — 스토어 등록에 필요한 스크린샷·문안이 준비된다.
선행: —관련MVP화면
S5-5기기 / 접근성 QADESIGNM
다양한 화면·설정에서 레이아웃·가독성 점검.
  • 기기 사이즈별 레이아웃, 글자 크기 확대, 대비
  • 다국어(MVP는 ko), 오버플로우 회귀 확인
완료 — 소형~대형 기기·큰 글자에서 레이아웃이 깨지지 않는다.
선행: —관련MVP화면
S5-6프로덕션 인프라 배포OPSL
로컬 도커를 벗어나 실제 서비스가 도는 환경 구성.
  • MySQL·Redis 매니지드, core-api 컨테이너/PaaS 배포
  • 시크릿·환경설정, 도메인·HTTPS, 헬스체크/롤링 배포
완료 — 프로덕션 URL로 앱이 붙어 전 기능이 동작한다.
선행: —관련MVP
S5-7크래시 / 분석OPSS
출시 후 문제를 잡고 사용을 이해할 최소 관측성.
  • Sentry or Firebase Crashlytics 연동
  • 핵심 이벤트 로깅(로그인·방생성·경주완료)
완료 — 크래시가 대시보드에 수집되고 핵심 퍼널이 계측된다.
선행: —관련MVP
S5-8개인정보 정책 + 심사 고지OPSM
스토어 심사·법적 요건. 특히 위치 데이터 처리 고지.
  • 개인정보 정책 호스팅(공개 URL), 위치데이터 처리방침
  • 앱 심사 데이터·권한 사용 목적 고지(App Privacy/데이터 안전)
완료 — 정책 URL이 공개되고 심사용 개인정보·권한 고지가 채워진다.
선행: S1-11관련MVP
S5-9서명 / 프로비저닝 + TestFlight 베타OPSM
실제 배포 채널로 빌드를 올려 베타 → 스토어 제출.
  • 서명·프로비저닝(iOS)·키스토어(Android)
  • TestFlight·내부 테스트 배포, 피드백 → 스토어 제출
완료 — 베타 빌드가 테스터에게 배포되고 스토어 심사 제출이 가능하다.
선행: S5-3, S5-6관련MVP
