Step 4 · 점수 · 랭킹 · 기록

경주 종료 → 4축 점수 → 랭킹·기록 누적. 보상 루프. (총 9 이슈)

📋 MVP Step 4 ← 이슈 보드
S4-1ScoringModule (4축 · 서버 권위)BACKENDM
경주 1건의 점수를 서버가 확정. 클라는 표시만.
  • 4축 — 등수300·거리200·완주220·여유100 (≤1000)
  • 미완주자 처리(완주보너스 0, 거리·등수 비례). 배점 밸런싱은 이후, MVP는 초안
완료 — 동일 경주 입력에 대해 서버가 일관된 4축 점수를 산출한다.
선행: S3-6관련MVP도메인
S4-2RaceResult 저장 (RaceFinished 구독)BACKENDM
경주 종료 이벤트를 받아 유저별 결과를 원천 데이터로 저장.
  • RaceFinished 구독 → 유저별 완주여부·거리·완주시간·등수·점수 저장
  • 멱등(중복 이벤트에도 1건), pointsAwarded 계산(적립은 MVP 미노출이어도 값은 계산 가능)
완료 — 경주가 끝나면 참가자 수만큼 RaceResult가 정확히 저장된다.
선행: S4-1, S3-6관련MVP도메인
S4-3RunnerStats 누적 + 티어BACKENDM
전체 누적 지표와 티어. 프로필·랭킹의 기반.
  • totalScore·totalDistanceKm·raceCount·winCount 누적 갱신
  • 티어 계산(브론즈~다이아, 컷 기준 임시)
완료 — 경주 종료마다 내 누적 통계·티어가 갱신된다.
선행: S4-2관련MVP도메인
S4-4랭킹 (전체 / 주간 / 월간)BACKENDM
누적 + 롤링 윈도우 랭킹. 명시적 Season 테이블 없이 날짜 윈도우로.
  • 전체 누적(RunnerStats) + 주간/월간(RaceResult 날짜 윈도우 or Redis ZSET)
  • 내 순위·주변 순위 조회
완료 — 전체/주간/월간 랭킹이 올바른 기간으로 집계된다.
선행: S4-2관련MVP도메인
S4-5랭킹 / 기록 / 결과 APIBACKENDS
앱 화면이 소비할 조회 엔드포인트.
  • GET /rankings?scope=, GET /me/results, GET /races/:id/result
완료 — 결과·랭킹·기록 화면이 필요한 데이터를 한 번에 받는다.
선행: S4-2, S4-3, S4-4관련MVP도메인
S4-6결과 화면 실연동APPS
경주 직후 시상대·순위·점수 분해를 실데이터로.
  • 시상대·전체 순위·점수 4축 분해(완주/DNF) — GET /races/:id/result
완료 — 경주를 마치면 실제 순위·점수가 결과 화면에 표시된다.
선행: S4-5관련MVP화면
S4-7랭킹 실연동APPS
전체/주간/월간 랭킹과 내 순위·티어를 실데이터로.
  • 세그먼트 전환 → GET /rankings?scope=, 내 랭킹 카드·티어 뱃지
완료 — 세그먼트별 랭킹과 내 위치가 실제 데이터로 나타난다.
선행: S4-5관련MVP화면
S4-8기록 / 프로필 통계 실연동APPS
지난 경주 히스토리와 누적 통계를 실데이터로.
  • 지난 경주 리스트(GET /me/results) + 상세, 프로필 누적 거리·경기·승률
완료 — 기록·프로필에 내 실제 경주 이력과 통계가 반영된다.
선행: S4-5관련MVP화면
S4-9mock 데이터 제거APPS
스캐폴딩용 mock을 걷어내고 전 화면을 실 repository로 통일.
  • data/mock.dart 등 제거, 전 화면 실 repository 연결 확인
완료 — 앱 어디에도 mock 경로가 남지 않고 모든 데이터가 서버에서 온다.
선행: S4-6, S4-7, S4-8관련MVP
