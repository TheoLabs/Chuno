Step 2 · 방 & 실시간 로비

방 생성·참가와 실시간(socket.io) 로비. 예약 시각 대기까지. (총 10 이슈)

📋 MVP Step 2 ← 이슈 보드
S2-1Room 애그리거트 + Participant + 마이그레이션BACKENDM
한 번의 경주를 위한 모집 단위. 목표·시작시각·정원과 참가자를 담는 일관성 경계.
  • Room(hostUserId·name·RaceGoal·max·scheduledStartAt·RoomStatus) + Participant(userId·joinedAt·isHost)
  • RaceGoal(targetKm·limitMinutes)·RoomStatus VO, RoomRepository
  • TypeORM 매핑·마이그레이션
완료 — Room 저장·조회 시 participants가 애그리거트 단위로 로드된다.
선행: S1-2, S1-3관련MVP도메인
S2-2Room APIBACKENDM
방 생성·탐색·참가·나가기·삭제. 불변식을 서비스에서 강제.
  • POST /rooms, GET /rooms(거리/시간/빈자리 필터), POST /rooms/:id/join·leave, DELETE /rooms/:id(방장·모집중)
  • 불변식 — 정원 초과 join 불가, 시작 후 join 불가, 최소 2명
완료 — 목록/생성/참가/나가기/삭제가 동작하고, 불변식 위반 시 명확한 에러.
선행: S2-1관련MVP도메인
S2-3예약 스케줄러BACKENDM
예약 시각에 방 상태를 자동 전환. 인원 미달이면 취소. 동시출발의 트리거.
  • BullMQ 지연잡 or @Interval — scheduledStartAt 도달 시 RECRUITING→STARTING→LIVE
  • 2명 미만이면 CANCELLED (경주 미성립)
  • 전환 시 도메인 이벤트 발행(RoomStarting 등)
완료 — 예약 시각에 자동 전환·LIVE 진입, 미달 방은 취소 처리된다.
선행: S2-1, S2-2관련MVP도메인
S2-4로비 socket.io 게이트웨이BACKENDM
로비 참가자에게 입퇴장·상태전환을 실시간 브로드캐스트.
  • 방 룸 조인/이탈, participantJoined·participantLeft·roomStatusChanged 브로드캐스트
  • JWT 핸드셰이크 인증
완료 — 한 클라이언트의 참가/나가기가 같은 방 다른 클라이언트에 즉시 반영.
선행: S2-1관련MVP도메인
S2-5Room 도메인 이벤트 (인프로세스)BACKENDS
컨텍스트 간 연결을 위한 이벤트 발행. MVP는 인프로세스 EventBus로.
  • RoomCreated·ParticipantJoined·RoomStarting·RoomCancelled 정의·발행
  • 구독자 배선(로비 게이트웨이·이후 알림)
완료 — 방 상태 변화가 이벤트로 발행되고 구독자가 반응한다.
선행: S2-1관련MVP도메인
S2-6홈 방목록 실연동 + 필터APPM
mock 목록을 실 API로 교체하고 필터·카운트다운을 실데이터로.
  • GET /rooms 연동, 거리/제한시간/빈자리 필터, 상태 뱃지(모집중/곧시작/LIVE)
  • 시작까지 카운트다운, 빈 상태 처리
완료 — 홈에 실제 방 목록이 뜨고 필터가 서버 결과에 반영된다.
선행: S2-2, S1-7관련MVP화면
S2-7방 생성 실연동APPS
생성 폼 입력을 실 API로 보내고 성공 시 로비로 이동.
  • 이름·거리·제한시간·인원·시작시각 → POST /rooms
  • 성공 시 생성된 방 로비로 pushReplacement
완료 — 폼으로 방을 만들면 서버에 생성되고 로비로 진입한다.
선행: S2-2관련MVP화면
S2-8로비 실시간APPM
socket.io로 참가자 입퇴장을 실시간 반영하고 나가기/방삭제 연동.
  • socket.io 연결·방 조인, 입퇴장 이벤트 → 참가자 그리드 갱신
  • 나가기(leave)·방삭제(방장) 연동, 방 취소 안내
완료 — 다른 사람이 참가/이탈하면 내 로비 화면이 즉시 갱신된다.
선행: S2-4관련MVP화면
S2-9서버시계 동기 카운트다운APPS
공정한 동시출발을 위해 클라 로컬시계가 아닌 서버 시각 기준으로 계산.
  • 서버 시각 오프셋 보정, elapsed = now − scheduledStartAt
  • STARTING 진입 시 카운트다운 화면 전환
완료 — 여러 기기의 카운트다운이 서버 기준으로 거의 동시에 0이 된다.
선행: S2-3관련MVP화면
S2-10방 엣지 상태 UXDESIGNS
인원 미달 취소, 빈 목록 등 예외 상황의 안내 화면.
  • 인원미달 취소 안내(환불/재모집 유도), 목록 빈 상태
완료 — 방이 취소되거나 목록이 비었을 때 사용자에게 명확히 안내된다.
선행: S2-6관련MVP화면
